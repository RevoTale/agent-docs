# Awesome Registry
This registry defines enforced utilities and libraries by stack or capability.

Rules:
- Read all matching awesome files before introducing, replacing, or removing libraries.
- Use `required` entries for matching capabilities.
- Any deviation requires explicit user `Accept` and must be documented as a strict-rule exception.

| key | awesome_file | scope |
| --- | --- | --- |
| go | ./go.md | Go runtime, backend services, Go templating, and Go GraphQL client |
| auth | ./auth.md | Authentication, authorization, identity, OAuth2, OIDC, and policy enforcement |
| react | ./react.md | React repositories and applications |
| nextjs | ./nextjs.md | Next.js repositories and applications |
